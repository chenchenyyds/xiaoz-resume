# 小Z简历 - 客服退款处理 SOP

> 标准化退款处理流程,保证一致性、避免纠纷。

---

## 一、退款总原则

| 情况 | 原则 | 处理人 |
|------|------|--------|
| 未使用未过期 | 全额退 | 客服直接处理 |
| 已使用 | 不退(特殊情况审批) | 客服发起,运营审批 |
| 系统故障 | 全额退 + 补偿 | 客服直接处理 |
| 试用积分 | 不退 | - |
| 兑换码积分 | 不退 | - |

---

## 二、流程图

```
用户发起退款
   ↓
客服受理(检查订单状态)
   ↓
  / 是未使用?  → 客服直接退(后台调 /api/admin/orders/:no/refund)
  \ 否
   ↓
  / 是系统故障? → 调监控查日志 → 确认后全额退
  \ 否
   ↓
  / 是已使用(部分)? → 拒绝 + 推荐"重新生成"
  \ 否
   ↓
特殊审批(运营)
   ↓
结果通知用户
```

## 三、Step-by-Step 操作

### 3.1 未使用未过期 - 标准退款

**触发**:用户买了次卡/月卡/增购,还没用任何积分,30 天内

**操作步骤**:
1. 后台 → 订单管理 → 找到订单(按手机号或订单号)
2. 检查:
   - 订单状态 = paid
   - 用户积分账户对应类型的余额 = 订单原始积分(未消耗)
   - 积分未过期
3. 全部满足 → 点"退款" → 确认
4. 后台自动:
   - 订单状态 → refunded
   - 积分账户余额归零
   - 调用虎皮椒退款接口
5. 通知用户:1-3 个工作日到账

**回复模板**:
> 亲,您的订单 [订单号] 退款已受理 ✅
> 
> 退款金额:¥[金额]
> 退款方式:原路返回
> 预计到账:1-3 个工作日
> 
> 感谢您的支持 ❤️

### 3.2 系统故障 - 全额退款

**触发**:用户反馈"充值没到账"、"重复扣费"、"改写失败但扣了积分"

**操作步骤**:
1. **不要立即退** — 先核实
2. 收集证据:
   - 用户手机号
   - 订单号(若有)
   - 故障截图
3. 后台检查:
   - 订单状态是否异常
   - 积分流水是否重复
   - 改写记录是否真的失败
4. 核实属实 → 调监控/日志确认
5. 退款 + **额外补偿 100 积分**(道歉诚意)
6. 通知用户

**回复模板**:
> 亲,非常抱歉给您带来困扰 🙏
> 
> 经核实,[具体故障描述]
> 
> 我们已经:
> 1. 全额退款 [金额] (1-3 工作日到账)
> 2. 额外补偿 100 积分(已到账)
> 
> 再次感谢您的理解 ❤️

### 3.3 已使用 - 拒绝退款

**触发**:用户积分已部分或全部使用

**操作步骤**:
1. 后台 → 用户 → 积分账户
2. 看到用户的 `total_used > 0`
3. 礼貌拒绝 + 推荐"重新生成"或"加购"

**回复模板**:
> 亲,您的订单 [订单号] 积分已使用 [N] 分
> 
> 根据《退款政策》,已使用积分不予退款
> 
> 但我们可以为您:
> ✅ 免费重新生成 1 次([链接])
> ✅ 转给人工顾问 1v1 沟通(工作日)
> 
> 请问您希望哪个方案?

### 3.4 特殊情况 - 审批退款

**触发**:用户强烈不满 / 投诉 / 有特殊情况(失业、生病等)

**操作步骤**:
1. 客服在后台记录特殊情况描述
2. 提交运营审批(微信/邮件)
3. 运营 24 小时内回复
4. 按运营决议执行

**审批邮件模板**:
> 退款审批 - [手机号]
> 
> 订单:[订单号] [金额]
> 积分:已使用 [N]/[M]
> 特殊情况:[描述]
> 用户诉求:[描述]
> 客服建议:[描述]
> 
> 请审批:同意/不同意/其他

## 四、争议处理

### 4.1 用户投诉"AI 效果差"

**场景**:用户改完不满意,要退款

**处理**:
1. **先看具体输出**:后台查看改写记录
2. **客观评价**:
   - 明显错误(乱码/空内容)→ 免费重新生成 + 必要时退款
   - 主观评价(改得不够好)→ 不退,推荐人工顾问

**回复模板**:
> 亲,改写效果不满意非常抱歉 🙏
> 
> 我们查看了您的改写记录:
> - 改写功能:[feature]
> - 输出长度:[N] 字符
> - 改写质量评估:[好/中/差]
> 
> 我们可以为您:
> ✅ 免费重新生成 1 次(我们已优化 prompt)
> ✅ 上传 JD 后重新改(更精准)
> ✅ 安排人工顾问沟通
> 
> 请问您希望?

### 4.2 用户投诉"被重复扣费"

**场景**:虎皮椒回调异常,同一订单多次扣积分

**处理**:
1. 后台查 `point_transactions` 该订单的 `related_id`
2. 看到重复扣分 → 全额退 + 补偿

**关键 SQL**:
```sql
-- 查某订单的积分流水
SELECT * FROM point_transactions
WHERE related_id = [订单 ID]
ORDER BY created_at;
```

### 4.3 用户投诉"推广返佣没到"

**场景**:邀请的人已支付,推广员没收到返佣

**处理**:
1. 后台查 `orders` 表,看 `invite_user_id` 是否正确
2. 查推广员的 `point_transactions`,看 source = 'commission' 的记录
3. 漏发 → 手动补发(`/api/admin/users/:id/adjust-points`)

## 六、退款数据统计

**每月统计**:
```sql
-- 月退款总额
SELECT
    DATE_TRUNC('month', refunded_at) AS month,
    COUNT(*) AS refund_count,
    SUM(refund_amount) AS refund_total
FROM orders
WHERE status = 'refunded'
  AND refunded_at >= NOW() - INTERVAL '3 months'
GROUP BY 1
ORDER BY 1 DESC;

-- 退款率
SELECT
    (SELECT COUNT(*) FROM orders WHERE status = 'refunded')::float /
    NULLIF((SELECT COUNT(*) FROM orders WHERE status = 'paid'), 0) AS refund_rate;
```

**V1 失败信号**:退款率 > 30% → 立即分析,可能 AI 改写质量有大问题
